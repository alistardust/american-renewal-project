# System Rules

System rules are cross-domain rules that apply across multiple pillars and prevent inconsistency, loopholes, structural drift, or domain-by-domain contradiction.

## Current rule families

- **Foundational rules** - mission, architecture, and baseline values
- **Geographic equality rules** - core rights and services should not fragment by location
- **Federalism-balance rules** - structure systems according to their main risk: fragmentation or capture
- **AI rules** - prevent asymmetry, denial bias, hidden harm, and accountability laundering
- **Domain-rule requirement** - each major pillar should define domain-specific rules consistent with the overarching ones

## Current emphasis

The branch chats repeatedly elevate three ideas to system-rule level:

1. Harmful decisions must involve accountable human judgment before harm occurs.
2. Absence of algorithmic approval must never count against a person.
3. Every pillar should inherit and implement the shared cross-domain rule layer.

## Canonical tracking

Formal rule IDs are tracked in the catalog database under `data/policy_catalog.sqlite`. The source chats currently include both top-level rules such as `SYS-FND-001` and domain-rule IDs such as `HLT-COV-003` and `JUS-AI-006`.
