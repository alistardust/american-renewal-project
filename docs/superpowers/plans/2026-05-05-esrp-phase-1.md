# ESRP Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce the five remaining Phase 1 deliverables that establish ESRP as a functioning research project (the charter is already complete and committed).

**Architecture:** Dependency-ordered — workspace setup first (templates and directory structure), then values mapping (short, clarifies the research frame), then techno-georgism memo (ESRP's distinctive intellectual contribution), then mechanism catalog entries (repeatable research units). All substantive outputs go through the adversarial review loop before being marked `reviewed`. Nothing gets marked `validated` without human review.

**Research methodology note:** Where software plans use TDD (write a failing test before writing code), ESRP uses adversarial-first drafting — read the adversarial review prompt *before* drafting each section so you are writing against the objections from the start rather than retrofitting a defense.

**Tech Stack:** Markdown (all research outputs), Claude / GPT-4o / Gemini (adversarial review), Elicit / Semantic Scholar (literature search), Zotero (citation management), Git (version control)

## The Adversarial Review Prompt

Every substantive output goes through this prompt, submitted to Claude, GPT-4o, and Gemini separately. Record the responses. For each objection: either revise to address it, or document explicitly why you chose not to.

> Identify the strongest objections to this from a mainstream economics perspective. Identify the strongest objections from a free-market perspective. Identify failure modes or unintended consequences the author may have missed. Identify empirical claims that require stronger evidence. Identify logical gaps or inconsistencies.
